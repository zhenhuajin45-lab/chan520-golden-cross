# A 股微观结构处理

## 对应评审项

- P1-1：涨跌停、T+1、次新、停牌。
- P1-2：数据覆盖率与失败告警。
- P2-1：量能口径说明。

## 涨跌停

`chan520_skill.microstructure.price_limit` 按代码识别涨跌停阈值：

| 板块 | 代码前缀 | 阈值 |
| --- | --- | ---: |
| 主板 | 60/000/001/002/003 | 10% |
| ST/*ST | 任意 | 5% |
| 创业板 | 300/301 | 20% |
| 科创板 | 688/689 | 20% |
| 北交所 | 8/4 | 30% |

`is_limit_up` / `is_limit_down` 默认使用 `limit - 0.3%` 的容差，并要求收盘价接近最高/最低价，用于过滤封板不可成交。

## T+1 与收盘信号

回测中信号在 D 日收盘后确认，最早 D+1 成交。若 D+1 买入涨停封板，跳过该笔买入；若 D+1 卖出跌停封板，顺延卖出。

## 次新/样本不足

`analyze` 默认 `--min-history 250`。有效 K 线不足时，不再静默跳过长期过滤，而是在 `defects` 中标注：

```text
历史数据不足 250 根K线，长期过滤降级，按次新/样本不足处理
```

## 停牌/缺 bar

目标日期无 bar 时，`trim_to_date` 抛出 `DataError`。扫描流程将其计入失败/覆盖率统计，不会插值。

## 数据质量

`data_quality` 检查：

- 最少 K 线数量。
- 日期去重且严格递增。
- OHLC 非 0/负/NaN。
- `high >= low`。
- `high >= max(open, close)`。
- `low <= min(open, close)`。

## 量能口径

原字段 `volume_ratio` 保留兼容，但报告中改称“5日放量倍数”。默认口径为：

```text
今日成交量 / 前5日平均成交量
```

可用 `--vol-base incl_today` 恢复旧口径：

```powershell
python -m chan520_skill analyze 600288 --date 2026-07-07 --vol-base incl_today
```

它不是交易所或行情软件盘中“量比”的严格定义，不能直接混同为盘口实时量比。
