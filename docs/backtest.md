# 回测说明

## 对应评审项

- P0-1：事件驱动回测与期望值验证。
- P1-1：D+1 执行、涨跌停不可成交、交易成本。
- P2-4：信号与执行价格的复权口径必须分离。
- Review v2：共享账户组合回测、风险预算、行业 cap、市场 regime 仓位、四不做、移动/时间止损。
- Review v3：修复 1 手塌缩、真实指数 regime、真实 `analyze()` 入选信号、板块层收窄、延迟移动止损、抗尾部统计。
- V4：D+1 开盘因果时钟、信号/执行价格分离、双边费用账本、可审计成交明细、确认型520与历史股票池快照接口。

## 实战版组合引擎

`backtest` 默认走 `portfolio_backtest_symbols`，不是逐票独立回测后简单相加。组合层只有一个共享资金账户：

- 初始资金由 `--initial-cash` 指定，默认 `100000`。
- D 日收盘标记权益并产生订单意图；D+1 开盘先执行上一日挂单，开盘仓位和熔断只读取 D 日已知状态。
- D+1 收盘后才更新持仓市值、日/周风控和下一交易日信号。
- 每个成交写入 `fills_*.csv`；每笔完整交易的 `costs` 包含全部建仓费用与平仓费用。
- 同一天多只股票触发时，用计划现金和计划行业/总仓位预占额度；D+1 开盘成交前再按实际开盘价复核一次 cap。
- 组合 equity curve 按交易日记录现金加持仓市值。

纪律默认值见 [risk.md](risk.md)，行业层说明见 [sector.md](sector.md)。

## 撮合规则

回测逐 bar 推进，任意日期 D 的信号只使用 `date <= D` 的 K 线。信号在 D 收盘后生成，最早在 D+1 执行。V4 禁止在 D+1 开盘成交、仓位上限或熔断判定中读取 D+1 收盘价或 D+1 指数状态。

默认成交价为 `next_open`，可选：

```powershell
python -m chan520_skill backtest 600288 --start 2026-01-01 --end 2026-07-01 --fill next_open
python -m chan520_skill backtest 600288 --start 2026-01-01 --end 2026-07-01 --fill next_vwap_proxy
```

`next_vwap_proxy` 使用 `(open + high + low + close) / 4` 近似。

## 入场纪律

组合回测只接受标准 520 入选信号。默认过滤：

- `MA5` 上穿 `MA20`。
- 收盘价站上 `MA5` 和 `MA20`。
- `MACD DIF > DEA`，且 MACD 金叉发生在520交叉前后2个交易 bar 内。
- `RSI14 <= 72`，避免过热追高。
- 指数 regime 必须为 `trend_up`，组合回测默认 `000300`。
- 行业层默认不作为入场否决，仅保留行业 cap；`--use-sector` 才启用代理行业宽度门控。
- 四不做必须通过：弱信号不做、急涨急跌不做、止损过远不做、`R:R < 2` 不做。
- 回测入选判定使用真实 `analyze()` verdict，不再使用 `_portfolio_signal` 替身；标准520可在金叉当天或金叉后1至3日确认期入选，前提是收盘持续站上 MA5/MA20。
- V4 默认信号与成交均使用不复权日线，避免端点前复权带来的历史信息；成交、仓位、费用、整手和涨跌停判断始终使用不复权日线。经点时审计的独立信号序列可注入研究流程，止损和目标在 D 日按比例映射至执行价格。

生成买入信号后，实际买入仍要经过风险预算：

```text
shares = floor(equity * 1% / (entry - stop) / 100) * 100
```

再叠加首仓 15%、单股 20%、行业 40%、总仓位 80/50/30、现金保留 20% 和整手约束。目标规模不足 1 手时跳过，不兜底买 1 手。

## 交易成本

默认参数：

| 成本项 | 默认值 | 方向 |
| --- | ---: | --- |
| 佣金 | 0.00025 | 双边，最低 5 元 |
| 印花税 | 0.0005 | 仅卖出 |
| 过户费 | 0.00001 | 仅沪市双边 |
| 滑点 | 5 bps | 双边 |

## A 股执行约束

- 买入执行日若涨停封板，则跳过买入。
- 卖出执行日若跌停封板，则顺延到下一根 bar。
- 停牌不插值；没有目标日 bar 时不会生成成交。
- 涨跌停判断使用 D+1 开盘价和前收盘价，不用 D 日收盘价提前假设可成交。

## 卖出纪律

持仓每日更新移动止损。触发以下任一项时，生成 D+1 卖出：

- 跌破移动止损；移动止损只有盈利达到 4% 后才上移到 `MA10` 或 `close - 2 * ATR14`。
- 入场后 7 个交易 bar 仍未达到 2% 浮盈。
- 持仓后连续 3 个 bar 不创新高。
- 跌破 `MA20`，或收盘确认达到保守第一目标位。

## 指标定义

| 指标 | 定义 |
| --- | --- |
| total_return | 累计净收益 / 初始资金 |
| CAGR | 按回测时间折算年化收益 |
| expectancy | 胜率 * 平均盈利 - 败率 * 平均亏损 |
| win_rate | 盈利交易笔数 / 总交易笔数 |
| payoff_ratio | 平均盈利 / 平均亏损 |
| profit_factor | 总盈利 / 总亏损 |
| max_drawdown | equity curve 峰谷回撤 |
| sharpe | 日收益 Sharpe，无风险利率按 0 |
| exposure | 持仓天数 / 有效回测天数 |
| trimmed_expectancy_5pct | 去掉前后 5% 尾部后的每笔期望 |
| median_trade_pnl | 每笔盈亏中位数 |
| expectancy_ci95_low/high | bootstrap 95% 置信区间 |
| sample_sufficient | 交易笔数是否达到 100 笔结论门槛 |

## 已知限制

- V4 默认拒绝行业映射缺失的入场，历史全市场回测应通过 `--universe-snapshot` 提供对应时点的股票池、ST/停牌资格和行业字段。
- 公开源未提供完整公司行为现金/送转明细；跨除权持仓必须单独审计。
- 在不少于200笔严格样本外交易、双倍成本和D+2延迟压力测试完成前，不得宣称策略具有正期望或可实盘部署。
