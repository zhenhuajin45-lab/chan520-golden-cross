# 策略入场、仓位风控与模拟盘缺口复盘

本文用于对齐当前 `chan520-golden-cross` 的真实能力边界，并和本机参考项目 `每日选股信号/macmini_daily_signal_core_work/strategy_lab/paper_trader.py` 的模拟盘规则做差距对照。结论只用于研究和本地模拟盘审计，不构成投资建议。

## 当前入场逻辑

### V4 标准 520 组合回测

V4 仍是最容易解释的基线策略，信号在 D 日收盘后生成，最早 D+1 开盘成交：

- `MA5` 上穿 `MA20`，且收盘价站上 `MA5`、`MA20`。
- `MACD DIF > DEA`，且 MACD 金叉发生在 520 交叉前后 2 个交易 bar 内。
- `RSI14 <= 72`，避免过热追高。
- 指数 regime 默认为 `trend_up` 才允许入场；行业门控默认不否决，只保留行业 cap。
- 四不做必须通过：弱信号不做、急涨急跌不做、止损距离过远不做、`R:R < 2` 不做。
- 信号与成交均使用不复权日线；D+1 成交、涨跌停、费用和整手判断不读取 D+1 收盘信息。

### V5 Alpha / 趋势回踩研究框架

V5 把 520 金叉降级为趋势确认，核心入场条件在 `_analyze_v5_candidate_signals`：

- 股票必须在动态股票池 `eligible_by_date` 中。
- 市场状态不能是 `BEAR`。
- Alpha 排名分 `ranking_score = alpha.total + sector_heat_bonus`，要求 `>= 70`。
- `trend_pullback_entry` 入场分要求 `>= 60`。
- 行业映射不能缺失；缺失时作为硬拒绝，fallback 状态会记录但不一定否决。
- 四不做、止损有效、`R:R >= entry_config.min_rr`、仓位可买到至少一手。
- 每个候选会输出 `reason_codes` 和 `reasons`；本地核心计划已自动把理由、止损、目标、R:R、价格区间和 T+1 风险缓冲传入下单账本。正式 V5/GM 路径仍保持关闭。

## 当前仓位与风控

仓位由 `RiskConfig` 和 `allowed_new_position_shares` 统一约束：

- 单笔风险预算默认 1%。
- 首仓资金上限 15%，最多一次加仓 5%。
- 单股上限 20%，单行业上限 40%。
- 最低现金保留 20%。
- regime 总仓位上限：`trend_up/BULL` 80%，`range/NORMAL` 50%，`down/BEAR` 30%。
- 建仓股数取风险预算、单股 cap、行业 cap、总仓位 cap、现金保留、分档仓位 cap 的最小值，再按 100 股整手下取整。
- 加仓只允许已有盈利、价格高于止损、趋势未破，并满足 20 日新高放量或回踩 5/10/20 日线止跌放量。
- 组合级风控包含最大回撤 15% 熔断、单日亏损 1.5% 次日停手、单周亏损 4% 后下周仓位乘数 0.5。

退出纪律当前偏日线收盘确认：

- 收盘跌破移动止损：未盈利 4% 前不抬止损，激活后按 `MA10` 或 `close - 2 * ATR14` 上移。
- 入场 7 个交易 bar 后仍未达到 2% 浮盈，触发时间止损。
- 持仓后连续 3 个 bar 不创新高。
- 收盘跌破 `MA20`。
- 达到保守目标位后收盘确认卖出。
- A 股执行层：买入执行日若涨停封板跳过；卖出执行日若跌停封板顺延。

## 本地模拟盘当前能力

本地模拟盘是离线 SQLite 账本，不连接真实券商：

- 默认资金 100 万。
- 手工 `buy/sell` 仍是明确的人工账本入口；自动路径使用“盘前计划 + 行情触发 + 二阶段确认”，不会按日线计划价直接成交。
- 校验 100 股整手、现金不足、持仓不足、`client_order_id` 幂等冲突。
- 记录 T+1 批次，带 `session_date` 的同日卖出会被拒绝。
- 工作台展示账户、持仓、每日汇总、订单和成交。
- 飞书推送盘前计划、成交和每日账户复盘卡片；估值不完整时盘后复盘停止推送。
- 本次补齐后，订单、成交、持仓、工作台和飞书都能展示 `signal_name`、`entry_reason`、`exit_reason`、`risk_reason`、`risk_reason_code`、`notes`。
- 本地模拟盘订单现在强制要求 `session_date`；缺失交易日会以 `SESSION_DATE_REQUIRED` fail-closed，避免 T+1 审计退化。
- `scripts/check_local_sim_readiness.py` 可在交易日前检查本地账本、dashboard、飞书 webhook、T+1 闸门、核心计划覆盖率、执行脚本、Python 版本和 `shadow_readiness=false`。
- `paper_runner close/open --local-sim-sync` 已能把 kernel 计划单和 open 成交同步到本地 SQLite 模拟盘；这是本地模拟盘 bridge，不等同于 GM shadow ready。

## 参考项目关键差异

参考项目的模拟盘更接近盘中执行工作流，值得迁移的规则包括：

- 严格价位触发区：5 分样本追价上限 6%，4 分样本 3%，弱信号 2%，过热样本 4%，并有 0.5% 下探容忍。
- 时间门：9:25 集合竞价门、9:35 开盘承接确认、10:30 触区回收确认、13:10 盘中/全球风险修复确认。
- 不倒推最低价成交：盘中低点触及买入区但当前未收回时，只等待，不用最低价假设成交。
- 全球风险仓位：防守/谨慎/基础/积极/机会总仓位约束为 45%/60%/75%/85%/92%，并按高贝塔敏感度调仓。
- 评分分层仓位：5 分基础 15%，4 分基础 10%，单票 cap 分别 18%/12%；过热、弱信号、全球风险继续降仓。
- 最小订单现金、一手缓冲、候选稀缺时优中取优提高备用金。
- 同标的重复加仓约束：已有浮亏、现价低于旧成本或超单票 cap 时不自动加仓。
- 同日风控回补禁买：同标的当日已风控卖出，不反手买回。
- T+1 风控卖出：开盘/当前价跌破失效价、硬止损 -5.5%、弱信号日 -4%、弱势续跌。
- 参考项目原规则是峰值浮盈超过 30% 后回吐 18%；本地中低频模拟盘改为更早的审计阈值：峰值浮盈达到 3%、回吐 1.5%，且保留利润比例低于 65% 时进入风险候选。

## 当前主要缺口

### 已完成的本地模拟盘闭环

- 盘前计划只使用上一交易日完整日线；计划按交易日过期，观察池 `WATCH_ONLY` 永不自动成交。
- 扫描覆盖率低于 85%、市场状态未知、报价日期/时间不新鲜、停牌、涨跌停、非连续竞价、触发区失效、板块急跌或相对弱势时 fail-closed。
- 买入执行采用两阶段确认，至少等待 2 分钟；确认走弱或超过 20 分钟会重置。计划价格几何不满足 `止损 < 下沿 <= 触发 <= 上沿 < 失效价` 时禁止进入可执行状态。
- 仓位风险按技术止损与 T+1 波动缓冲的较大值计算；主板缓冲底线 7.5%，创业板/科创板 12%，并结合 ATR 与平均振幅继续上调。
- 风险扫描支持 MA20 失效、MA5 弱势续跌、-5.5% 硬止损和 3% 峰值/1.5% 回吐利润保护，多理由合并到稳定的单标的风险计划；未完成计划跨交易日保留 `armed_at`、首次确认报价和确认状态。
- 风险卖出执行器先检查 T+1、报价新鲜度、跌停可成交性；硬止损使用单次新鲜报价快速通道，MA20/MA5 失效和利润回吐继续使用二阶段确认。同日卖出回补、前一交易日风控卖出回补和亏损加仓均被 broker 拒绝。
- 风险循环 readiness 与新增买入 readiness 已拆分；扫描覆盖不足或没有可执行核心计划时，仍可自动管理已有持仓，但不允许开新仓。
- 09:30-09:44 每两分钟运行一次开盘风险循环；账户存在未完成风险卖单时先禁止新开仓，两个及以上主要指数跌幅达到 1.5% 时触发盘中广度冲击门。
- 账户总仓位按现价估值；实时行情失败时仅使用最后有效报价并标记 `STALE_CACHE`，没有缓存则估值不完整并停止盘后复盘推送。
- 成本口径已与回测统一为佣金 0.025%、最低 5 元、卖出印花税 0.05%、沪市过户费 0.001%。
- 工作台展示计划状态、扫描覆盖率、T+1 可卖数量、估值来源、股票名称和红绿盈亏；飞书分为计划、成交和复盘三类卡片。
- 每笔计划和成交记录策略版本、结构化理由、止损、目标、R:R、T+1 风险缓冲、计划仓位和实际成交价。当前核心计划版本为 `local_sim_core_plan_v2`，风险版本为 `local_sim_risk_policy_v2`。

### 仍保持关闭的正式能力

- 本机没有 GM 数据底座，开源全市场扫描不是正式 V5 动态股票池、行业热度和 sector map 的等价替代。正式 `auto_open_close_kernel_ready` 继续为 `false`。
- GM adapter 尚未完成正式 open/close kernel 的回放、对账和异常演练，`gm_adapter_shadow_ready` 与 `shadow_readiness` 继续为 `false`。
- V11 条件随机化证据没有证明 alpha ranked 选择增益，因此本地自动执行禁用 alpha 排名和评分放大仓位，采用硬门后代码顺序与固定 5% 基础仓位。
- 开源行情覆盖率必须达到 85% 才允许新增买入。2026-07-15 扫描覆盖率为 55.69%，所以 2026-07-16 的正确计划是零新增买入，仅执行现有持仓风控。

## 下一步研究顺序

1. 提高开源日线扫描稳定性并保存逐源失败分类，使全市场覆盖率稳定达到 85% 以上。
2. 为开源模式补充可审计的历史行业映射和动态股票池，之后才允许声明 V5 候选等价。
3. 用扩展样本重新验证 alpha 排名；在统计证据转正前不恢复评分排序和评分放大仓位。
4. 最后完成 GM adapter 回放、对账、断网/重复提交/部分成交演练；通过前不改变 `shadow_readiness=false`。
