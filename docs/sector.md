# 行业与板块层

## 对应评审项

- Review v2：材料中的“大盘-行业-个股”三层共振不能只靠个股均线替代。
- Review v2：组合回测必须有行业 cap，避免同一题材内重复下注。

## 当前实现与第三轮收窄

`chan520_skill.sector` 提供三个层次：

- `DEFAULT_SECTOR_MAP`：内置常用样本股行业映射，覆盖本轮回测证据篮子。
- `industry_of`：把代码映射到行业，未命中时归入 `未知行业`。
- `sector_state_from_members`：用同一行业内样本成员计算行业状态。

行业状态目前用公开日线可稳定计算的代理指标：

```text
sector breadth = 行业内收盘价站上 MA60 的成员数 / 有效成员数
sector 60d rise = 行业内成员 60 日涨幅均值
sector_ok = breadth >= 55% 且 sector 60d rise >= 0
```

这不是申万行业指数的精确复刻，但比完全没有行业层更接近实战框架：个股 520 信号必须获得行业宽度和 60 日方向确认。

第三轮后，组合回测默认 `use_sector=False`，也就是不再把该代理行业宽度作为入场否决条件。原因是当前样本中大量行业只有 1-2 个成员，宽度会退化成个股自身条件，不能宣称真实“三层共振”。

## 回测中的门控

默认组合回测入场需要同时满足：

- 指数市场 regime 为 `trend_up`。
- 个股标准 520 入选。
- 单行业持仓市值不超过权益的 40%。

只有显式传入 `--use-sector` 时，才启用当前代理行业门控：

```powershell
python -m chan520_skill backtest --basket "..." --start 2024-07-01 --end 2026-07-01 --use-sector
```

D 日生成候选时会先按计划仓位预占行业额度；D+1 执行时再用实际开盘价复核行业 cap。

## 后续升级

全市场部署建议补两层缓存：

- 行业归属缓存：从东方财富、腾讯或本地静态表生成 `code -> industry`。
- 行业指数/行业成分缓存：优先使用真实行业指数或完整行业成分，而不是仅用回测篮子成员做代理。

V4 默认拒绝行业映射缺失的入场，不再把所有未映射标的归入同一个“未知行业”风险桶。全市场历史回测应通过 `--universe-snapshot` 提供对应时点的行业字段。

在真实行业指数与成分缓存完成前，当前系统只能称为“指数大盘层 + 个股信号 + 行业风险 cap”，不能称为完整三层共振。
